Git

  • Version Control System.
  • Manage Source Coode History.
  • Local Tool.
  • Repository: location where your code history is stored.
  • Branch: "Folder" within the repository containg commits.
  • Commits: different stages of your code inside a branch.

    • Commands

    $git init

    Initializes a local Git repository/ creates a repository.

    $git status

    To see the status of the Git repository if we have any untracked changes.

    $git add .

    To add all files/ changes.

    $git commit -m "..."

    Commit a code.

    $git branch

    To see branches that are available.

    $git checkout -b ...

    Create new branch and after -b should be written a branch name.

    $git log

    Allows to get overview of all the commits that we have in the branch that we are currently located in.

    $git checkout ...

    You can use it to do both, check out branches or in that case, commits. And when doing git log, you will get the ID of a specific commmit and then get checkout plus ID to check out this commit.

    git checkout -- .

    Helps to go back to the last commit we did and to get rid of all unstaged changes.

    $git master

    Change branch name for master branch.

    $git reset --hard ...

    Resets the index and working tree. Any changes to tracked files in the working tree since commit are discarded. Instead of ... should be the ID of the community we want to go back.

    $git merge ...

    To merge some branch and after merge should be written branch name.

    $git branch -D ...

    To delete a branch and after -D should be written branch name as well.

    GitHub

  • Hosting and Collaboration Provider.
  • Git Repositry Hosting.
  • Web Based.

    • Commands

    $git remote add origin ...

    To establish a connection between the local repository and a remote repository on GitHub. Origin is the name for the remote repository. We want to make sure that we establish the connection between a specific remote repository. And here you have to add a link that was created on GitHub when you made a new repository.

    $git remote

    List the remote connections you have to other repositories.

    $git remote -v

    List the current remotes associated with the local repository./ To see the origin more specifically, url behind that.

    $git push -u origin master

    To push your branches to the remote repository; -u means that it basically establishes an upstream between Git and GitHub.

    $git branch

    To see in which branch we are currently.

    $git clone ...

    To clone/ copy someone's repository to our computer. Here should be added the link too. /Clone (download) a repository that already exists on GitHub, including all of the files, branches, and commits.

    $git status

    Always a good idea, this command shows you what branch you're on, what files are in the working or staging directory, and any other important information.

    $git push

    To push the files when it's already connected to the repository. Uploads all local branch commits to the remote.

    $git fetch

    To synchronize the code in GitHub repository and to remote tracking branch.

    $git merge origin/master

    To merge the local repository with the remote tracking branch.

    $git pull

    Updates your current local working branch with all new commits from the corresponding remote branch on GitHub. git pull is a combination of git fetch and git merge.

    $git remote rm origin

    Remove the connection to the remote repository.
