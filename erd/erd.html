Table Room

  • Descriptions of what the table means and how it's functionally used (in the UI, etc.): there can be lots of rooms in one serie.
  • Descriptions of what each attribute means: Id is an unique value and a primary key (int) and RoomName include names of rooms (varchar(55) means that RoomName can hold until 55 letter and/or numbers)
  • Explanations of the relationships (foreign keys) from this table to others, and vice-versa: RoomId is the primary key in the Room table (unique value) and it's a foreign key in the Series table. So it's quite easy to get to know which Room is connected to the Serie (e.g when the RoomId is 1, we know that it equals to "Kitchen" etc.

Table Series

  • Descriptions of what the table means and how it's functionally used (in the UI, etc.): there can be lots of rooms in one serie.
  • Descriptions of what each attribute means: Id is an unique value and a primary key (int), SeriesName include names of rooms (varchar(55) means that SeriesName can hold until 55 letter and/or numbers) and RoomId is a foreign key from Room Table.
  • Explanations of the relationships (foreign keys) from this table to others, and vice-versa: SeriesId is a primary key in the Series table (unique value, in other words: each SerieName equals to one unique SerieId); RoomId is a foreign key from the Room table.
